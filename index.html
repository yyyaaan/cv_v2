My Data Science Workflow

Capture, Extract, Load, Transform, Analyze, Report | Both ETL and ELT model

  • in my work...
  • in my hobby project...

In existing workflow orchestration, compile job can be done with one-click and submitted as BASH/Perl. However, it has strong limitations and tough assumptions to be accomplished. My development/improvement project fulfills the “grey area” that existing workflow could not cover.

Source: varied from Excel, CSV, TSV, SAS to direct or adapted DB access.

Tools: BASH (workflow orchestration, version control), Python without 3rd library (process monitoring, CSV/TSV pre- and post-processing, etc.), SQL (transformation, standardizing, etc.), Oracle DB/SAS (final data loading) .

Analytics: regulatory reports, advanced statistical models in R or SAS.

My hobby project fulfills my tech-mind in terms of modern cloud service, and also serve my need for interesting data.

Concept: no-limitation for the cloud and myself in terms of programming languages or unfamiliar areas.

Source: mostly captured webpages (NodeJS/Puppeteer), scheduled streaming and traditional ETL.

Tools: Google Cloud, incl. Virtual Machine (Unix, as my hub), BigQuery (database), Cloud Function (severless, in NodeJS and Python), DataStudio/BI (dashboard), R-Shiny server (dashboard). Plus Git, Bash and miscellaneous network configuration and cloud infrastructure installation. Separate Storage service is not used for cost concerns.

Analytics: push notifications to my phone, chatbot, dashboard with interactive plotting; occasionally model fitting in Tensorflow-Python (preferred over scikit-learn). Examples in Kaggle
